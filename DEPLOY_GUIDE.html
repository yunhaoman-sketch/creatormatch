① 推送代码到 GitHub

1

在 GitHub 创建新仓库

访问 github.com/new，填写：

  • Repository name：creatormatch（或 globe-talent）
  • Visibility：Public（Vercel/Render 免费层需要公开仓库）
  • ❌ 不要勾选 "Add a README"（本地已有）

点击 Create repository，记住仓库 URL（如 https://github.com/yourname/creatormatch）

2

推送本地代码到 GitHub

打开终端，在 ai-hunter 目录下运行：

# 替换 yourname 为你的 GitHub 用户名
git remote add origin https://github.com/yourname/creatormatch.git
git branch -M main
git push -u origin main

如果要求认证，使用 GitHub 的 Personal Access Token（Settings → Developer settings → Personal access tokens）

② 部署后端到 Render

💡 Render 免费层：每月 750 小时（单服务足够），不活跃时会休眠（15分钟后首次请求较慢），适合 MVP/演示使用。
1

创建 Web Service

  • 登录 render.com → New + → Web Service
  • 选择 Build and deploy from a Git repository
  • 连接 GitHub，选择 creatormatch 仓库
2

配置部署参数

设置项填写值
Namecreatormatch-api
RegionSingapore（亚洲用户最快）
Branchmain
Root Directorybackend
RuntimePython 3
Build Commandpip install -r requirements.txt
Start Commandgunicorn app:app --bind 0.0.0.0:$PORT --workers 1 --timeout 120
Instance TypeFree
3

添加环境变量

在 Environment 选项卡添加：

变量名值必填
LLM_API_KEYsk-xxxx（你的 API Key）可选
LLM_API_BASEhttps://api.openai.com/v1可选
FRONTEND_ORIGIN部署后填写 Vercel URL可选
FLASK_ENVproduction可选

⚠️ LLM_API_KEY 不填也能运行，只是搜索用规则引擎而非 AI 解析

4

等待部署完成

点击 Create Web Service，等待约 3-5 分钟。

部署成功后，记录你的 Render URL，格式如：

https://creatormatch-api.onrender.com

验证：访问 https://creatormatch-api.onrender.com/api/health，应返回：

{"service": "CreatorMatch API", "status": "ok", "version": "1.0.0"}

③ 更新前端 API 地址

修改 frontend/index.html 中的后端 URL

打开 frontend/index.html，找到第 997 行附近：

// 将这行：
return 'https://creatormatch-api.onrender.com/api';

// 改为你实际的 Render URL：
return 'https://你的服务名.onrender.com/api';

修改后提交并推送：

git add frontend/index.html
git commit -m "fix: update backend API URL to Render deployment"
git push

④ 部署前端到 Vercel

1

导入 GitHub 仓库

  • 登录 vercel.com → Add New → Project
  • 选择 creatormatch 仓库，点击 Import
2

配置部署参数

设置项填写值
Framework PresetOther（不是 Next.js！）
Root Directoryfrontend（点击 Edit 修改）
Build Command（留空）
Output Directory.（留空或填 .）
3

完成部署

点击 Deploy，等待约 1 分钟。

部署成功后，Vercel 会分配一个 URL，格式如：

https://creatormatch.vercel.app
# 或
https://creatormatch-yourname.vercel.app
4

回到 Render 更新 CORS 配置

将 Vercel URL 填入 Render 的 FRONTEND_ORIGIN 环境变量：

Render Dashboard → 你的服务 → Environment → 添加/更新：

FRONTEND_ORIGIN = https://creatormatch.vercel.app

Render 会自动重新部署，精确 CORS 控制生效。

🌐 绑定自定义域名（可选）

Vercel 前端绑定域名

  • Project → Settings → Domains
  • 输入你的域名（如 creatormatch.com）
  • 按提示在你的域名注册商添加 CNAME 或 A 记录
  • Vercel 自动签发 SSL 证书

Render 后端绑定域名

  • Service → Settings → Custom Domain
  • 输入子域名（如 api.creatormatch.com）
  • 添加 CNAME 记录指向 Render 提供的域名
  • 更新前端中的 API URL 和 Render 的 FRONTEND_ORIGIN 环境变量
💡 免费域名推荐：Freenom（.tk/.ml/.ga 免费）或 js.org（for JS项目）。Vercel 子域名本身就是永久免费的。

🔐 环境变量汇总

变量名说明示例值设置位置

❓ 常见问题排查

🔴 Render 部署失败：ModuleNotFoundError

检查 requirements.txt 是否包含所有依赖。在 Render Logs 中查看具体缺少什么包，添加到 requirements.txt 后重新推送。

🔴 前端请求报 CORS 错误

1. 确认 Render 服务已成功启动
2. 确认 FRONTEND_ORIGIN 填写了正确的 Vercel URL（不含尾部斜杠）
3. 或临时将 FRONTEND_ORIGIN 留空（允许所有来源）进行调试

🟡 Render 冷启动慢（首次请求30秒）

免费层不活跃时会休眠。解决方案：
1. 接受冷启动延迟（仅第一次）
2. 使用 UptimeRobot 定时 ping API 保活
3. 升级到 Render Starter（$7/月，不休眠）

🔴 Vercel 部署后 404 空白页

确认 Root Directory 设置为 frontend（不是项目根目录）。在 Vercel Project Settings → General → Root Directory 中检查。

🟡 SQLite 数据在 Render 重启后丢失

Render 免费层文件系统是临时的，每次部署/重启后 hunter.db 会重置。
若需持久化：迁移至 Neon（免费 PostgreSQL） 或 Render PostgreSQL（免费层提供）。

🔴 邮件发送失败

Gmail 需开启"两步验证"后使用"应用专用密码"，不能使用账号密码直接登录。QQ邮箱需在邮箱设置中开启 SMTP 并获取授权码。

🗄 升级为持久化数据库（推荐生产环境）

⚠️ Render 免费层的 SQLite 数据会在重启后清空。如需正式使用，建议升级为 PostgreSQL。

使用 Neon 免费 PostgreSQL（5分钟完成）

  1. 访问 neon.tech 注册，创建免费数据库
  2. 复制 Connection String（格式：postgresql://user:pass@host/db）
  3. 在 Render 环境变量添加 DATABASE_URL
  4. 在 requirements.txt 添加 psycopg2-binary
  5. 修改 app.py 的 get_db() 函数，检测 DATABASE_URL 决定使用 SQLite 还是 PostgreSQL
